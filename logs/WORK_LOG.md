### TREE 更新流程与趋势图防复发规则细化

触发原因：
- `0703V1` 交付后，用户发现 `数据折线图` 区域出现 `系列1` 和点位数字重叠，说明此前“图表存在、锚点正确”的审计不够，缺少对图表显示元素和视觉结果的硬性验收。

已固化到规则文件：

- 明确以后每次 TREE 更新必须按“识别输入、复核日报、保护 TREE 结构、同步数据、更新写入时间、更新趋势解析、最终审计”的顺序执行。
- 明确趋势图更新不是复制日报图，而是用 TREE 当前值同源的底层历史序列更新 TREE 图表缓存。

详细工作步骤与对应规则：
1. 识别输入文件。
   - 步骤：从根目录识别最新权威 TREE 和最新手动刷新日报。
   - 规则：不能只看修改时间；必须复核 workbook 结构、可见 sheet、表头、行布局、指标代码位置和 xlsx 完整性。

2. 复核日报数据源。
   - 步骤：确认 `宏观数据`、`指数走势`、`辅助数据` 等底层 sheet 存在，并抽查近期非空观测。
   - 规则：例行 TREE 更新默认不跑 `refresh_daily.ps1`，不更新日报趋势图；日报只作为用户手动刷新后的底层数据源。

3. 保护 TREE 结构。
   - 步骤：按最新 TREE 的实际结构更新，简化版保持简化版，复杂版保持复杂版。
   - 规则：只更新目标主表；非目标 sheet 需用 worksheet XML 对比确认未改动。隐藏 sheet 状态不得被意外改变。

4. 同步 TREE 数据。
   - 步骤：从同一底层历史序列同步当前数据、数据日期、环比、同比、同比增量、宏观颜色和 TREE 趋势图缓存。
   - 规则：优先按 Wind/source code 精确匹配；找不到的编码行必须核对月报/补充源，不能用相似名称替代，不能写 0。

5. 更新 `写入时间`。
   - 步骤：比较新版与上一版的当前值和数据日期。
   - 规则：只有当前值或真实数据日期变化才刷新 `写入时间`；Excel 序列号、`datetime` 和日期文本必须归一成同一个日历日期再比较；输出只保留日期级 `yyyy-mm-dd`，默认不写小时、分钟或秒。

6. 更新趋势解析文字。
   - 步骤：读取与 TREE 当前值、趋势图相同来源的底层历史序列，判断近期方向、最新边际变化和当前所处区间。
   - 规则：不能只看前台当前值、单个环比值或图形外观下结论；底层序列缺失或口径不确定时保留谨慎说明或留空，并在审计里标记。

7. 更新 TREE 趋势图。
   - 步骤：只更新图表 value cache，并保留原锚点、尺寸和红色单折线样式。
   - 规则：更新缓存后必须清理所有可能显示文字的图表元素，包括 `c:dLbls` 数据标签、点位标签、系列名标签、图例、拥挤坐标文字、标题、网格线、填充和边框；不能依赖模板里的隐藏标签继续隐藏。

8. 图表 XML 审计。
   - 步骤：全量扫描目标主表所有趋势图 XML。
   - 规则：趋势图数量必须等于预期；锚点必须仍在趋势图列；`total_dLbls=0`；`total_legend=0`；有效来源行的图表缓存点数必须大于 0。

9. 视觉渲染审计。
   - 步骤：渲染或打开包含趋势图列的代表性区域，至少覆盖前 50 行和本次图表修复涉及区域。
   - 规则：必须确认肉眼可见结果为干净红色小折线，没有 `系列1`、点位数字、图例、坐标文字重叠或空白图。若渲染因权限失败，要申请本地写图权限重跑；不能在未渲染时声称“视觉已通过”。

10. 交付前总审计。
    - 步骤：检查 xlsx 完整性、sheet 数量和隐藏状态、非目标 sheet XML、公式错误、日期格式、写入时间缺失、趋势解析空白与趋势图一致性、图表 XML 和视觉预览。
    - 规则：任一强制项失败时，必须生成修正版并重新审计，不得把半通过版本作为最终 TREE 交付。

脚本层防复发：

- 明确不适合计算同比增速的指标：净投放/净回笼、利率、利差、收益率、PMI、准备金率、杠杆率、赤字率等比例或扩散类指标，同比列可填 `-`，保留同比增量或其他更有解释力的变化口径。
- 明确颜色规则：仅对宏观数据行使用宏观颜色逻辑，A股、港股、美股、宽基指数、估值、情绪和中国产业等资本市场或产业行不默认套用。当前值看近一年历史分位，高位红、低位绿，近一年基本不变不填色；环比看近一年自身环比变化幅度，只有显著变化才填色，正红负绿；同比看近一年同比分位，高红低绿；同比增量看近一年自身同比增量幅度，显著正向红、显著负向绿。
- 明确颜色含义：颜色只表示该指标在自身历史中的位置或变化幅度，不直接等同于“好/坏”；分析时应先看有颜色且权重较高的指标，再结合指标含义判断宏观结论。
