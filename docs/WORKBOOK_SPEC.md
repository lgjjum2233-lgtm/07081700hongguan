- `A股港股`
- `中债`
- `中国宏观`

共同列结构：

| 列 | 字段 | 说明 |
| --- | --- | --- |
| A | 指标名称 | 指标展示名称。 |
| B | 频率 | 日度、周度、月度、季度、不定期等。 |
| C | 代码 | Wind 指标代码或自定义辅助代码。 |

| H | 上期数据 | 上一期数据。 |
| I | 上期日期 | 上一期数据日期。 |
| J | 趋势图 | 无坐标小折线图。 |

## 宏观分析汇总工作簿

当前根目录未保留现行宏观分析汇总表；历史版本见 `archive/old_summary_versions/`。

核心工作表：`中国宏观分析汇总`

关键列：

| 列 | 字段 | 说明 |
| --- | --- | --- |
| I | 监测指标 | 汇总表中的指标名称，用于匹配日报指标。 |
| L | 发布频率 | 指标更新频率。 |
| O | 当前数据 | 从日报匹配得到。 |
| P | 数据日期 | 从日报匹配得到。 |
| Q | 边际变化 | 从日报匹配得到或按底层序列计算。 |
| R | 26 年数据趋势图 | 从日报趋势数据或底层序列生成。 |

注意：TREE 表中 A股、港股等股市指数的边际变化使用日涨跌幅口径，即 `今日收盘 / 昨日收盘 - 1`；不要使用点位差。不同 TREE 版本列位会变化，后续脚本必须按表头定位。

注意：TREE 权重理由列只保留解释本身，不保留 `权重较高，因为`、`权重中等，因为`、`权重较低，因为` 等前缀。

注意：TREE `中国基本面` 中 `产业` 部分不纳入权重合计；产业行权重占比用 `-` 表示，其余中国基本面指标需要重新归一至 100%。

当前中国基本面分组权重口径：流动性面 `42%`、政策面 `8%`、增长 `27.78%`、价格利润 `22.22%`、产业 `-`。

当前 TREE 主表参考文件：`（0706V1）TREE宏观分析最新版.xlsx`。该文件为 0703V2 按新流程同步 7月6日日报后的精简交付版，仅保留可见主表和复核表；主表图表 XML 已审计通过。

当前 TREE 主表关键字段：

| 字段 | 说明 |
| --- | --- |
| 权重理由 | 指标在框架中的角色、权重依据和解释，不保留旧权重前缀。 |
| 定义 | 专业口径定义；当前已逐行复核底层代码和概念边界。 |
| 怎么用的（重点） | 指标上行、下行或异常变化的判断方法。 |
| 发布频率 | 指标更新频率。 |
| 指标代码 | Wind 代码、自定义组合代码或 `—`。 |
| 当前数据 | 从刷新后日报缓存同步得到或按保护规则保留。 |
| 数据日期 | 当前数据对应日期。 |
| 写入时间 | 本地 TREE 写入/捕获日期，只保留 `yyyy-mm-dd`；不是官方发布日期或 Wind 后台发布时间。 |
| 环比 | 与上一期相比的变化；股市指数使用日涨跌幅。 |
| 同比 | 同比读数或按底层序列计算的同比变化；不具备经济意义的比率类指标写 `-`。 |
| 同比增量 | 当前值相对去年同期的增量；同比指标表示百分点变化。 |
| 数据折线图 | 26 年数据趋势图。 |
| 趋势解析 | 必须基于刷新后日报或经核验的月报/补充底层历史序列判断，不只看前台缓存值。 |

## 匹配规则

优先级：

1. 汇总表指标行号到日报前台行号的显式映射。
2. 汇总表指标行号到 `宏观数据` 或 `指数走势` 行号的显式映射。
3. 对利差类指标，使用底层两条收益率序列相减计算。
4. 无匹配则填 `—`。

当前匹配脚本：`scripts/match_daily_to_macro_summary.ps1`

## 图表规则

- 趋势图使用红色无坐标单折线图。
- 宏观分析汇总表历史版本通常把趋势数据写入隐藏列 AA 以后，并在 R 列放置嵌入式图表对象。
- TREE 趋势图只更新 TREE 自身图表缓存，不直接复制日报趋势图。
- TREE 趋势图必须清理数据标签、系列名、点位标签和图例；XML 审计要求 `total_dLbls=0`、`total_legend=0`。
- 图表修改后必须做代表性区域视觉验证，确认没有标签乱码、文字重叠或空白图。
