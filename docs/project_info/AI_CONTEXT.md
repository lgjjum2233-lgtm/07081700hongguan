# AI 接手上下文

本文件给后续 AI 模型快速理解项目使用。请优先阅读本文件，然后阅读 `docs/AI_WORKFLOWS.md`、`docs/project_info/README.md`、`docs/WORKBOOK_SPEC.md` 和 `logs/WORK_LOG.md`。

## 项目目标

构建一个每日更新的宏观日报和宏观分析汇总体系，用于跟踪：

- A 股和港股市场数据
- 中债收益率和利差数据
- 中国宏观指标
- 后续计划新增的美国和全球宏观指标

数据主要通过 WPS 表格顶部 `Wind` 插件刷新。刷新后，日报中的数据会匹配到宏观分析汇总表，更新当前数据、数据日期、边际变化和趋势图。

## AI Workflow 入口

重复性工作请先读 `docs/AI_WORKFLOWS.md`：

- 更新 TREE 表时，按 `docs/ai_workflows/TREE_UPDATE_WORKFLOW.md` 执行，并继续遵守 `tools/tree_daily_update/TREE_UPDATE_RULES.md`。
- 输出宏观日报结论、辅助判断表或 PPT 结论时，按 `docs/ai_workflows/MACRO_CONCLUSION_WORKFLOW.md` 执行。
- 这两个 workflow 是给 GitHub 和后续 AI 模型读取的稳定操作手册；`logs/WORK_LOG.md` 和对应工作日志保留完整历史细节。

## Excel 逻辑

日报文件主要工作表：

- `A股港股`：市场指数、成交额、换手率、南北向资金等。
- `中债`：国债收益率、期限利差、信用利差等。
- `中国宏观`：货币政策、社融、信贷、财政、地产、PMI、工业企业利润等。
- `宏观数据`：Wind 底层数据源，含指标历史序列。
- `指数走势`：部分市场价格或指数历史序列。
- `辅助数据`：趋势图辅助数据。

日报前台表核心列：

- A：指标名称
- B：频率
- C：代码
- D：单位
- E：当前数据
- F：数据日期
- G：变化
- H：上期数据
- I：上期日期
- J：趋势图

宏观分析汇总表核心列：

- I：监测指标
- L：发布频率
- O：当前数据
- P：数据日期
- Q：边际变化
- R：26 年数据趋势图

TREE 当前主表核心列：

- I：权重理由
- J：定义
- K：怎么用的（重点）
- L：发布频率
- M：重点跟踪项
- N：指标代码
- O：当前数据
- P：数据日期
- Q：写入时间
- R：环比
- S：同比
- T：同比增量
- U：数据折线图
- V：趋势解析

## 脚本说明

正式脚本在 `scripts/` 下。

- `match_daily_to_macro_summary.ps1`：把日报数据匹配到宏观分析汇总表，更新 O/P/Q/R 列，并生成趋势图。
- `expand_daily_macro_v30.ps1`：把 V30 新增指标接入日报 `宏观数据`，并扩展展示页和 `海外数据` sheet。
- `add_12_macro_indicators.ps1`：将新增 12 个中国宏观指标加入日报。
- `fix_trends_openxml.ps1`：修复趋势图引用和图表结构。
- `fix_trend_row_heights.ps1`：修复新增指标行高，使趋势图显示一致。
- `verify_wps_open_summary.ps1`：用 WPS/表格 COM 打开并验证汇总表可读、图表对象存在。

`codex_tmp/` 是临时工作区，不应作为正式项目代码来源。
`tools/tree_daily_update/` 是新流程日常更新工具目录，包含日报刷新、图表缓存修正、TREE 同步和审计复核脚本。
`archive/` 存放旧版本和参考输入，具体分类见 `docs/WORKSPACE_INDEX_V1.md`。

## AI 操作约定

- 不要覆盖原始文件。
- TREE 更新和宏观结论输出属于已固化 workflow，动手前先读取 `docs/AI_WORKFLOWS.md` 及对应 workflow 手册。
- 新文件必须带版本号，例如 `_V1.xlsx`、`_V1.1.xlsx`、`_V2.xlsx`。
- 操作 Excel 前，先确认当前最新版本文件。
- 例行 TREE 更新默认使用用户手动刷新后的最新日报；不自动刷新日报，也不更新日报前台趋势图，除非用户明确要求。
- 涉及 Wind 刷新时，优先通过 WPS 顶部 `Wind -> 刷新数据` 操作。
- 涉及图表修改后，必须用 WPS 打开验证。
- TREE `写入时间` 只记录本地写入/捕获日期，格式为 `yyyy-mm-dd`，不能当作官方发布日期或 Wind 后台发布时间。
- TREE 权重理由列只保留解释部分，不写 `权重较高，因为`、`权重中等，因为`、`权重较低，因为` 等前缀。
- TREE 中国基本面中 `产业` 部分不纳入权重合计；调整权重时产业行保持 `-`，其他中国基本面指标重新归一至 100%。
- 中国基本面政策面偏定性，不宜过高；当前建议保持政策面低于流动性面，优先让高频流动性指标承担更高代表权重。
- 日报仍未接入 `SPX.GI`、`IXIC.GI`、`DJI.GI` 的 Wind 缓存序列；TREE 第 112-114 行沿用既有外部补数逻辑，后续如日报接入这些序列应优先使用 Wind 缓存。
- 修改项目后，更新 `logs/CHANGELOG.md` 和 `logs/WORK_LOG.md`。
- 输出宏观日报结论时，必须基于最新 TREE 和日报重建辅助判断表；若 TREE 当前数据与旧 `指标判断`、趋势解析或旧 PPT 文案冲突，以当前数据和日报底层复核为准，并在辅助表标记待复核。
