# TREE 更新 Workflow

本文件是给后续 AI 模型使用的 TREE 表更新工作手册。详细规则仍以 `tools/tree_daily_update/TREE_UPDATE_RULES.md` 为准；本文件负责把日常执行顺序、验证口径和交付说明压缩成可快速读取的 runbook。

## 目的

把用户已刷新后的最新日报底层数据，同步到最新权威 TREE 表中，并更新：

- 当前数据
- 数据日期
- 写入时间
- 环比、同比、同比增量
- 宏观颜色
- TREE 自身趋势图
- 趋势解析文字

最终交付的新 TREE 必须保持输入 TREE 的结构、sheet、样式、图表锚点和非目标 sheet 不被意外改变。

## 权威来源

优先读取以下文件：

1. `tools/tree_daily_update/TREE_UPDATE_RULES.md`
2. `logs/WORK_LOG.md`
3. `docs/project_info/AI_CONTEXT.md`
4. `docs/WORKBOOK_SPEC.md`

其中 `TREE_UPDATE_RULES.md` 是计算、格式、图表和审计规则的最高优先级文件。

## 输入识别

例行 TREE 更新需要两个输入：

- 最新权威 TREE workbook。
- 最新手动刷新后的日报 workbook。

如果用户明确指定文件，使用用户指定文件。否则从工作区根目录识别候选文件，但不能只看文件修改时间。必须复核：

- xlsx 压缩包完整。
- TREE 可见 sheet、隐藏 sheet、主表名、表头和行布局合理。
- 日报包含必要底层 sheet，例如 `宏观数据`、`指数走势`、`辅助数据`。
- 日报底层 sheet 有近期非空观测值。
- 代表性前台数据没有明显滞后于底层数据。

## 默认规则

- 例行 TREE 更新默认不运行 `refresh_daily.ps1`。
- 例行 TREE 更新默认不刷新日报前台趋势图。
- 日报只是用户手动刷新后的底层数据源。
- TREE 趋势图必须从日报或补充源的底层历史序列单独更新，不能直接复制日报趋势图。
- 如果用户明确要求 Codex 刷新日报，再执行日报 Wind 刷新流程。

## 标准流程

1. 识别最新权威 TREE 和最新日报。
2. 复核日报数据源。
3. 保护 TREE 结构。
4. 按 Wind/source code 精确匹配底层序列。
5. 同步当前数据、数据日期、环比、同比、同比增量和宏观颜色。
6. 根据当前数据或真实数据日期是否变化，更新 `写入时间`。
7. 基于同一底层历史序列更新趋势解析文字。
8. 更新 TREE 趋势图 chart value cache。
9. 清理趋势图所有可能显示文字的元素。
10. 运行交付前审计。
11. 记录工作日志。

## 数据与计算规则摘要

- 当前数据使用刷新后日报或补充源中的最新值。
- 数据日期必须使用真实观测日期，不能把 Excel 序列号直接作为文本展示。
- `写入时间` 是本地 TREE 写入或捕获日期，不是官方发布日期。
- 只有当前数据或真实数据日期变化时，才刷新该行 `写入时间`。
- `写入时间` 只保留到日期，格式为 `yyyy-mm-dd`；默认不写小时、分钟或秒。
- 股票指数行的边际变化使用涨跌幅，不使用点位差。
- 利率、利差、PMI、准备金率、杠杆率、赤字率等指标的同比增长率通常不具备经济意义，应写 `-`，但同比增量可以保留。
- 百分比、比率和增速类指标按百分号展示；普通数值不追加中文单位。
- 颜色规则必须按照历史分位或历史边际变化强度执行，不能凭主观判断上色。

## 趋势解析规则

趋势解析必须来自与当前值和趋势图相同的底层历史序列。不能只看：

- 前台当前值
- 单个环比值
- 图形外观
- 旧版趋势解析文字

趋势解析至少要判断：

- 最新方向。
- 最新变化相对近期历史是否显著。
- 当前值处于自身近期区间的高位、低位还是中性。
- 该指标上升或下降在经济含义上是偏正面、偏负面还是中性。

底层数据缺失、过旧、太稀疏或口径冲突时，应保留谨慎说明或留空，并在审计记录中标明。

## TREE 趋势图规则

TREE 趋势图必须是干净的小型红色单折线图。更新 chart cache 后必须清理：

- `c:dLbls` 数据标签。
- 点位标签。
- 系列名标签。
- 图例。
- 坐标轴拥挤文字。
- 标题。
- 网格线。
- 填充和边框。

不要依赖模板里隐藏的标签继续隐藏。`0703V1` 曾出现 `系列1` 和点位数字重叠，原因就是图表模板残留数据标签节点；后续必须把 XML 和视觉检查都纳入交付前审计。

## 审计清单

交付前至少检查：

- xlsx 压缩包完整。
- TREE 可见 sheet 数、隐藏 sheet 状态与输入结构一致。
- 非目标 sheet worksheet XML 未被意外改动。
- 主表表头、行顺序、指标代码位置正确。
- 明显公式错误数量为 0，或遗留错误已说明。
- 日期列显示格式正确，例如 `yyyy-mm-dd`。
- `写入时间` 不应在有效数据行缺失。
- `写入时间` 应为日期级 `yyyy-mm-dd`，不应带小时、分钟或秒。
- 有趋势图的有效数据行不应缺少趋势解析，除非有明确原因。
- 趋势图数量符合预期。
- 趋势图锚点仍在趋势图列。
- `total_dLbls=0`。
- `total_legend=0`。
- 有效来源行的图表缓存点数大于 0。
- 代表性区域视觉预览通过，图表没有 `系列1`、点位数字、图例、坐标文字重叠或空白图。

如果视觉渲染因权限失败，必须说明视觉审计未完成；不能只凭 XML 审计声称视觉已通过。

## 交付说明

最终回复至少说明：

- 使用的 TREE 输入文件。
- 使用的日报输入文件。
- 输出 TREE 文件。
- 数据更新行数。
- 趋势图更新数。
- 趋势解析更新数。
- `写入时间` 刷新行数。
- XML 审计是否通过。
- 视觉审计是否通过。
- 遗留问题或未匹配指标。

## 日志要求

每次完成后更新 `logs/WORK_LOG.md`，记录：

- 输入文件。
- 输出文件。
- 主要操作。
- 更新行数。
- 未匹配项。
- 审计结果。
- 遗留问题。

如果本次改变了项目对外文件口径或目录口径，也更新 `logs/CHANGELOG.md` 和相关索引文档。

## 常见错误

- 只按修改时间选择 TREE 或日报。
- 把旧 TREE 结构覆盖到新 TREE。
- 用相似指标名替代缺失 Wind code。
- 将缺失数据写成 0。
- 把日报趋势图复制到 TREE。
- 更新图表缓存后不清理数据标签。
- 未渲染视觉预览却声称视觉审计通过。
- 将 `写入时间` 误写成官方发布日期。
- 对没有经济意义的比率型指标计算同比增长率。
