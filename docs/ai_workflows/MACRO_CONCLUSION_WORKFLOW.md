# 宏观日报结论输出 Workflow

本文件是给后续 AI 模型使用的宏观日报结论输出工作手册。详细工作日志见 `logs/宏观日报结论输出工作日志_20260703.md`；本文件负责沉淀可重复执行的判断链条。

## 目的

基于最新 TREE 和日报底层数据，形成可复核、可追溯、可更新的宏观结论，并在用户确认后更新到 TREE 或宏观日报 PPT 中。

完整链条是：

```text
最新 TREE + 日报底层数据
-> 重建辅助判断表
-> 第一轮筛选指标
-> 面级综合判断
-> 反向证据检查
-> 第二轮筛选展示指标
-> 输出标准结论
-> 用户确认
-> 更新 TREE/PPT
```

## 数据源规则

- 最新 TREE 是本期框架和指标池的主来源。
- 日报表是底层复核数据源。
- 旧辅助表只能作为字段结构和格式模板，不能直接沿用判断。
- 旧 TREE、旧 PPT、旧辅助表不能作为本期判断数据源。
- 旧 PPT 可以作为样式参考，但旧结论不能直接迁移。
- TREE 的当前数据、数据日期、环比、同比、同比增量是本期判断的优先字段；TREE 中旧的 `指标判断`、趋势解析或面级文案只能作为提示，不能覆盖当前数据。
- 如果 TREE 当前数据与 `指标判断`、趋势解析或旧 PPT 文案冲突，以最新 TREE 当前数据和日报底层复核结果为准，并在辅助判断表中标记为口径冲突或旧判断待更新。

TREE 提供：

- 面级框架。
- 指标归属。
- 当前值、环比、同比、同比增量。
- 权重和指标判断。
- 哪些指标可以参与本期判断。

日报底层数据用于复核：

- 当前值。
- 变化值。
- 历史分位。
- 趋势位置。
- 序列变化。
- 阈值关系。

## 数据一致性与口径优先级

输出结论前必须先做数据一致性检查，避免用旧文字解释新数据。

检查顺序：

1. 识别最新 TREE 和最新日报，不能只按文件修改时间判断。
2. 从最新 TREE 抽取每个指标的当前数据、数据日期、环比、同比、同比增量和指标判断。
3. 用日报底层数据复核关键指标的当前值、变化值、分位数和阈值关系。
4. 比较 TREE 数据字段与 `指标判断`、趋势解析、旧辅助表、旧 PPT 文案是否一致。
5. 如果数字字段和文字判断冲突，数字字段和日报复核优先；旧文字不能进入最终结论。
6. 如果 TREE 和日报底层数据冲突，先核对指标代码、频率、日期、单位和口径；无法确认时标为 `待复核`，不进入第二轮 PPT 展示。

重点口径：

- 公开市场操作要区分日度净投放/净回笼和周度净投放/净回笼，不能用一个口径替代另一个口径。
- 资金面判断不能只看公开市场操作数量，还要结合资金价格和政策利率阈值，例如 DR007 与 7 天逆回购、R007 与 DR007。
- 债券定价数据要同步检查收益率当前值和 bp 变化方向，箭头方向必须与变化方向一致。
- 美国短端流动性要优先看 SOFR 与 IORB 的相对位置，再结合 TGA、ON RRP 等数量指标。
- 估值面必须同时给 PE/PB 等估值水平和近三年分位，不能只给绝对值。

## 辅助判断表定位

辅助判断表不是事后复核表，而是结论生成工作台。每次输出结论前都必须基于最新 TREE 和日报重新生成或重新核验。

辅助判断表需要回答三类问题：

1. 指标怎么看。
2. 指标是否进入第一轮候选池。
3. 指标是否进入第二轮 PPT 展示。

建议字段包括：

- 行类型、大类、资产/市场、面级模块、子维度。
- 原表行号、监测指标、数据频率/新鲜度。
- 当前值、数据日期、环比、同比、同比增量。
- TREE 数据与日报复核是否一致。
- `指标判断`/趋势解析是否与当前数据一致。
- 权重占比、指标判断、权重/重要性复核。
- 角色标签、方向判断、影响强度。
- 关键阈值/对照基准。
- 第一轮候选、第一轮入选逻辑。
- 反向证据检查。
- 面级综合判断、结论强度约束。
- 第二轮进PPT、展示优先级、第二轮入选逻辑。
- 最终展示口径、未进入PPT原因。
- PPT最终文案/面级输出。
- 待复核点。

辅助判断表中的 `指标判断` 字段不是直接结论。它需要经过数据一致性检查、第一轮筛选、反向证据检查和面级综合判断后，才能转化为最终判断词。

## 第一轮筛选

第一轮筛选判断的是：这个指标是否足以影响面级结论。

优先进入第一轮的指标包括：

- 权重高的核心指标，例如社零、固投、地产投资、非农、CPI、PPI、核心PCE、DR007、社融、成交额、两融、PE分位。
- 边际变化大的指标，例如环比大幅回落或明显改善。
- 突破关键阈值的指标，例如 PMI 是否高于 50、DR007 是否高于 7 天逆回购、SOFR 是否高于 IORB、估值分位是否处于近三年高位。
- 需要联动判断的指标，例如 DR007 与 7 天逆回购、SOFR 与 IORB、债市资金价格与国债收益率。
- 对宏观面级判断具有锚定作用的指标。

低频滞后指标通常作为背景。只有在明显异常或边际变化特别大时，才进入第一轮。

## 面级综合判断

面级结论必须由第一轮候选池综合得出，不能由最终展示的单一指标倒推。

综合判断时同时看：

- 权重。
- 当前值。
- 环比。
- 同比。
- 同比增量。
- 历史分位。
- 阈值关系。
- 指标之间是否互相印证。
- 是否存在反向证据。

判断词应反映综合证据强弱，例如：

- 中性。
- 中性偏弱。
- 中性偏宽松。
- 中性偏多。
- 边际收敛。
- 结构性支撑延续。

## 反向证据检查

输出结论前必须做反向证据检查，避免单边叙事。

常见检查：

- 基本面偏弱时，是否存在 PMI 或出口改善。
- 资金价格上行时，公开市场操作是否显示扰动缓和。
- A 股估值偏高时，港股科技估值是否仍相对中性。
- 债市基本面偏利多时，制造业景气是否改善。
- 美国需求判断是否同时覆盖就业、消费、景气和通胀。

有反向证据时，结论应收敛语气，例如用“内需承压，但制造业景气改善”，而不是过度悲观。

## 第二轮 PPT 展示筛选

第二轮筛选服务于 PPT 展示，不等同于完整判断依据。

原则：

- 每个面最多展示 1 到 3 组关键指标。
- 只展示最能解释结论的数据。
- 数据必须能直接支撑判断词。
- 括号里的数据要简洁。
- 能用边际变化表达的，不优先使用容易造成误解的大同比数据。

第二轮输出应保留未进入 PPT 的原因，避免后续误以为未展示指标没有参与判断。

## 标准结论句式

统一使用：

```text
面：判断词：核心逻辑：分项解释（对应数据）；分项解释（对应数据）；分项解释（对应数据）。
```

要求：

- 判断词来自面级综合判断。
- 核心逻辑来自第一轮候选池。
- 括号数据来自第二轮展示指标。
- 不堆砌所有指标。
- 不用无法支撑判断词的数据。

## PPT 更新规则

更新 PPT 时遵守：

- 不改变原 PPT 无关内容。
- 只更新宏观结论部分。
- 判断词加粗并标色。
- 普通解释文字不加粗。
- 上行箭头使用红色。
- 下行箭头使用绿色。
- “中性”偏灰色。
- 偏积极、偏强、偏多等偏正面判断使用红色。
- 偏高、偏空、偏谨慎等偏负面判断使用绿色。
- 若文字放不下，先缩小字号，再压缩文字。

更新 PPT 前应先让用户确认结论文本，避免把未确认判断直接写入交付稿。

## 特别注意事项

- 资金面不能轻易写“紧缩”。
- DR007 必须和 7 天逆回购比较。
- SOFR 必须和 IORB 比较。
- 公开市场操作日度和周度必须分开表述，不能混用。
- 当 `指标判断` 文案与当前数据冲突时，必须以当前数据和日报复核为准，并把旧文案标记为待更新。
- 美国基本面不能只看消费者信心等单一软指标。
- 估值面必须给近三年分位。
- 北向资金成交额不是净流入，只能说明外资参与度或交易活跃度。
- 股市和债市不能简单使用同一套指标。
- 最终 PPT 指标不是全部判断依据。

## 交付说明

最终回复至少说明：

- 使用的 TREE 文件。
- 使用的日报文件。
- 辅助判断表输出文件。
- 辅助判断表生成行数。
- 第一轮候选数量。
- 第二轮展示数量。
- 覆盖的面级结论数量。
- 本期结论摘要。
- 是否已更新 TREE。
- 是否已更新 PPT。
- 用户尚未确认或仍需复核的点。

## 日志要求

完成后更新 `logs/WORK_LOG.md`，记录：

- 输入文件。
- 输出辅助判断表。
- 第一轮和第二轮筛选数量。
- 面级结论摘要。
- 更新 TREE/PPT 的文件名。
- 样式和排版检查结果。
- 遗留问题。

如果本次改变了对外 PPT 或当前文件口径，也更新 `logs/CHANGELOG.md` 和相关索引文档。

## 常见错误

- 沿用旧辅助表判断。
- 直接迁移旧 PPT 结论。
- 只看最终展示指标来倒推面级判断。
- 不做反向证据检查。
- 把资金边际收敛写成紧缩。
- 把北向成交额写成净流入。
- 只看 PE 绝对值，不看近三年分位。
- 用股市资金面指标直接替代债市资金面判断。
- 未经用户确认就更新 PPT 交付稿。
